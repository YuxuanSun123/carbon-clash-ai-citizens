# DeepSeek AI 集成设置指南

本项目已集成 DeepSeek API 来控制游戏中的 AI 玩家，使其具备更智能的决策能力。

## 功能特性

### 🤖 智能 AI 玩家
- **商业 AI**: 专注于经济利益最大化，会根据当前局势智能选择最优的建筑投资策略
- **环保 AI**: 注重环境保护和可持续发展，优先选择环保建筑和政策

### 🧠 AI 决策能力
- **建筑选择**: AI 会分析当前金钱、环境状况、其他玩家情况等因素来决定建造什么建筑
- **政策选择**: 在遇到政策事件时，AI 会根据自己的角色特点选择最合适的政策选项
- **智能对话**: AI 的决策过程更加人性化和智能化

## 设置步骤

### 1. 获取 DeepSeek API 密钥
1. 访问 [DeepSeek 平台](https://platform.deepseek.com/)
2. 注册账号并登录
3. 前往 [API Keys 页面](https://platform.deepseek.com/api_keys)
4. 创建新的 API 密钥

### 2. 配置环境变量
1. 项目根目录已创建 `.env` 文件模板
2. 编辑 `.env` 文件，将 `your_api_key_here` 替换为您的实际 API 密钥：
```
VITE_DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 3. 重启开发服务器
**重要**: 修改环境变量后必须重启开发服务器才能生效
```bash
# 停止当前服务器 (Ctrl+C)
# 然后重新启动
npm run dev
```

## 使用说明

### AI 行为模式

#### 商业 AI (黄色方块)
- 优先建造工厂以获得高收入
- 在政策选择中倾向于经济利益
- 会分析投资回报率和风险
- 在资金充足时会考虑多样化投资

#### 环保 AI (绿色方块)
- 优先建造绿色建筑
- 避免高污染的工厂建设
- 在政策选择中优先考虑环保效益
- 会平衡经济发展和环境保护

### 备用机制

如果 DeepSeek API 调用失败（网络问题、API 密钥错误等），系统会自动降级到原有的简单 AI 逻辑，确保游戏正常进行。

## 技术实现

### 文件结构
```
src/
├── services/
│   └── deepseekApi.ts          # DeepSeek API 服务
├── hooks/
│   └── useMultiPlayerGameState.ts  # 游戏状态管理（已集成 AI）
└── ...
```

### API 调用流程
1. 游戏状态收集：收集当前玩家状态、其他玩家信息、游戏阶段等
2. 上下文构建：根据 AI 角色特点构建系统提示词
3. API 请求：发送请求到 DeepSeek API
4. 响应解析：解析 AI 的决策结果
5. 备用处理：如果 API 失败，使用原有逻辑

## 故障排除

### 常见问题

1. **AI 使用简单逻辑而非 DeepSeek**
   - 检查 `.env` 文件中的 API 密钥是否正确
   - 确认网络连接正常
   - 查看浏览器控制台是否有错误信息

2. **API 调用失败**
   - 验证 API 密钥是否有效
   - 检查 DeepSeek 账户余额
   - 确认 API 调用频率未超限

3. **环境变量未生效**
   - 确保 `.env` 文件在项目根目录
   - 重启开发服务器
   - 检查变量名是否以 `VITE_` 开头

### 调试信息

在浏览器控制台中，你可以看到以下调试信息：
- `DeepSeek AI (玩家名) 建筑选择: xxx` - AI 的建筑决策
- `DeepSeek AI (玩家名) 政策选择: xxx` - AI 的政策决策
- `DeepSeek建筑选择失败，使用备用逻辑` - API 调用失败时的降级提示

## 成本说明

DeepSeek API 按调用次数计费，每次 AI 决策会产生少量费用。为了控制成本：
- API 调用已优化，只在必要时调用
- 设置了合理的 token 限制
- 提供了备用逻辑确保游戏体验

建议在测试阶段监控 API 使用情况，根据需要调整调用频率。
